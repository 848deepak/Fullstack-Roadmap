# 00 — OOP & Modules Quick Reference

## Core Terms
- **Class**: Blueprint for objects.
- **Constructor**: Runs when object is created.
- **Encapsulation**: Hide internal details (`#private` fields).
- **Inheritance**: Child class extends parent class.
- **Module**: File-based code organization with `import/export`.

## When to Use
- Reusable models, domain entities, and scalable file architecture.

## Key Patterns
- Keep classes focused on one responsibility.
- Use modules for clean separation of concerns.
