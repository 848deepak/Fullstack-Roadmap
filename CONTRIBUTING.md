# Contributing Guide

Thank you for contributing to this learning repository.

## How to contribute

1. Fork the repository
2. Create a branch: `feature/<topic-name>`
3. Keep each pull request focused on one improvement
4. Add clear README notes if you add a new topic/project
5. Run project checks before opening PR

## Contribution ideas

- Add new JavaScript/React exercises
- Improve explanations and code comments
- Add test cases for existing modules
- Improve accessibility and error handling in mini projects

## Coding standards

- Use meaningful variable names
- Prefer readable code over clever shortcuts
- Add learning-friendly comments for non-obvious logic
- Keep files small and focused
